# StrataWP AI-Readiness System — Design Spec (Phase 4)

- **Date:** 2026-06-25
- **Branch:** `feat/ai-readiness` (off `main` @ `de6be0d`)
- **Status:** Draft — awaiting review (revised after adversarial review)
- **Sub-project 4 of the framework-hardening program.** Comparison target is codenamed **Triple XXX**.

## 1. Goal

Make StrataWP **agent-ready by merit, not marketing**: ship the AI capability neither framework has — a first-party **MCP server** that exposes StrataWP's deterministic generators and discovery as typed tools/resources — and an **enforced contract-first protocol** where machine-readable declaration files (`block.json`, `theme.json`, the MCP tool schemas) are the single source of truth, with types generated from them and **drift/violations gated in CI**.

North star (scorecard rematch): on the **AI-readiness** dimension StrataWP moves from "declares AI conventions in docs" to "ships an MCP server + schema-enforced contract codegen, both green in CI" — the same "declare → **enforce** with a green gate" pattern that made Phases 1–3 credible.

**Hard framing (non-negotiable, reconciles with the shipped v2.0 narrative):** this is **NOT** an embedded AI SDK. No hosted-LLM providers, no API keys, no "AI-powered generation." `ROADMAP.md:7,15,38` deliberately rejects a baked-in AI wrapper (the deleted `@stratawp/ai`); Phase 4 honors that. The MCP server is a **thin, deterministic protocol surface that the user's own editor agent** (Claude Code/Cursor/Copilot) drives. The published `@stratawp/mcp` product has **nothing to do** with the repo's `.mcp.json`/`.claude` claude-flow dev harness.

## 2. Current reality (verified by recon + adversarial review)

- **Greenfield, not catch-up.** No `@modelcontextprotocol` code exists in `packages/`; **neither StrataWP nor Triple XXX ships an MCP server.** The competitor's AI edge is inferred (agent-legibility), not a committed benchmark. The win is **shipping the gated capability**, not parity.
- **The deleted packages were not this.** `@stratawp/ai` (Phase 0) was a hosted OpenAI/Anthropic CLI wrapper — **do not revive.** `@stratawp/studio`'s REST read-shapes (theme.json/block/pattern) are the only legitimate salvage, not its React UI.
- **Capability surface to expose (verified):**
  - **CLI generators** `packages/cli/src/commands/{block,component,template,part}.ts` — plain `(name, options)` functions wired directly as Commander `.action()` handlers; **no interactive prompts inside them**. `component/template/part` already delegate string-generation to pure helpers in `packages/cli/src/utils/templates.ts`; **`block.ts` is the outlier** — its four generators (`generateBlockConfig/Edit/Render/Styles`) are inline, it hardcodes `name: 'stratawp/<slug>'` (namespace not parameterized), and (like the others) it interleaves `console.log`/`ora`/`process.exit(1)`/`fs` writes / cwd-resolution / "already exists" guards / namespace+slug inference-by-reading-the-filesystem with generation. **These four commands have no existing tests.**
  - **Explorer discovery** `packages/explorer/src/utils/component-discovery.ts` → `ComponentDiscovery.discoverAll(): Promise<ComponentInfo[]>` (typed in `packages/explorer/src/types.ts`), already served at `/api/components*`. Backs the read resources directly.
- **The enforcement gap _is_ Phase 4:** no `ajv`/`zod`/`json-schema`/`@modelcontextprotocol` anywhere in `packages/`; CI runs typecheck/lint/test/phpcs/phpstan/phpunit/axe/perf but **no contract/schema-validation step**. No explicit `zod` dependency (transitive floor 3.23.8).
- **Pre-existing `block.json` naming debt (load-bearing for WS-B):** ~**15 of 20** example/template `block.json` files violate a "name prefix = theme namespace" rule today — stale `forge-basic/*` (e.g. every `feature-card`) and cross-theme copy-paste (`strata-basic/hero` in advanced/store). apiVersion 3 ✅ and `render` present (all dynamic) ✅ everywhere. The strict gate therefore **requires fixing these names first** (incl. the `packages/cli/templates/**` copies) — mandatory cleanup, not an optional baseline. Blocks use `edit.tsx` + `index.tsx`; **none have `save.tsx`**.
- **Overclaims (coordinate, don't extend) — full truth-up is Phase 5:** `README.md:55` "Full type safety across PHP and JavaScript" (JS half partially earned once WS-B codegen ships), `:97` "52+ patterns", `:59` "PHP Hot Reload … without page refresh" (it's a full reload), stale `CHANGELOG.md:549` v0.4.0 AI entry (describes the deleted package — annotate in WS-A, decision 12).
- **Repo-fit nuance (corrected):** `pnpm-workspace.yaml` globs `packages/*`; `turbo.json` (Turbo **1.x** `pipeline`) fans out by **matching script name** — so `build`/`typecheck`/`test` reach a new package **only if that package declares those scripts** (e.g. `packages/explorer` has no `test` and is silently skipped). Root `lint` (`eslint .`) + `format:check` (`prettier --check "**/*.{ts,tsx,md,json}"`) are repo-globs and reach new files automatically. **A `git diff --exit-code` contract gate is NOT covered by any existing job — it needs an explicit `ci.yml` job.** `examples/**` and `packages/cli/templates/**` are eslint-ignored and have **no `typecheck` script** today.

## 3. Scope

**Option B.** Two workstreams, one spec, **one plan each, one PR each**.

- **WS-A — `@stratawp/mcp` server** first (the differentiator; rides existing CI for build/typecheck/test; adds the contract job).
- **WS-B — contract-first enforcement** second (validates WS-A's surface + the framework's declaration files; ships the block.json→TS codegen).

**Out of scope:** any embedded/hosted LLM or provider keys (explicitly excluded); `@stratawp/registry`; a `studio` React admin UI; headless `WordPressClient` data tools + the hooks/filter manifest (deferred); automated npm publish via changesets (manual `pnpm release`; decision 8); destructive/credentialed CLI actions as tools; unifying the divergent vite-plugin↔explorer `BlockMetadata` types (latent inconsistency, **not** needed — WS-A's resources use explorer's `ComponentInfo`, WS-A's tools use their own zod output schemas; deferred). Phase 5 (broad doc truth-up) stays separate.

## 4. Workstreams

### WS-A — `@stratawp/mcp` MCP server (PR #1)

1. **New package `packages/mcp`** (`@stratawp/mcp`, `bin: stratawp-mcp`), `tsup` cloned from `packages/cli` (esm, dts, clean, sourcemap, `shims:true`, banner, `external` the SDK). **Must declare its own `build`, `typecheck`, `test` scripts** (or turbo silently skips them); no `lint` script needed (root `eslint .` covers it). Deps: **`@modelcontextprotocol/sdk ^1.29.0`** — the **stable v1.x single package**; deep imports `@modelcontextprotocol/sdk/server/mcp.js`, `/server/stdio.js`, `/inMemory.js`. **NOT** the v2 alpha scoped packages (they import `zod/v4` and wrap `inputSchema` in `z.object()`). Add explicit **`zod ^3.25`** (SDK 1.29 declares `zod ^3.25 || ^4.0`); a single hoisted zod copy is essential — two copies break the SDK's `instanceof` checks (verify no dual-zod after install).
2. **Tools — refactor generator cores into PURE functions that RETURN values; never shell out, never exit, never write stdout.** For each command, extract a pure core (in `packages/cli`, e.g. `utils/templates.ts` or a `generators/` module) with signature roughly `(input) => { files: { path: string; content: string }[]; messages: string[] }` — it takes the **target dir + namespace + theme slug as explicit inputs** (no `process.cwd()`, no `console.*`/`ora`, no `process.exit`, no `fs` writes). The Commander layer keeps cwd-resolution, spinners, the "already exists" guard, namespace/slug inference, exit codes, and the actual file writes. (`block.ts` is the largest extraction — move its 4 inline generators out and **parameterize the block namespace**.) The MCP tools `scaffold_block`/`scaffold_component`/`scaffold_template`/`scaffold_part` call the pure cores and perform the writes themselves into the input dir. Each tool declares a zod **`inputSchema` (a ZodRawShape, not `z.object(...)`) + `outputSchema`**; derive TS types via `z.infer`. **If a pure core still needs to signal failure, it throws/returns an error result — it must never `process.exit` (that would kill the MCP server) or `console.log` (that would poison the stdio JSON-RPC stream).**
3. **Resources — read-only, from explorer discovery.** `list_components`, `get_component`, `get_component_source` backed by `ComponentDiscovery.discoverAll()`. No mutation.
4. **Transport + hygiene.** Stdio transport. **All logging to stderr.** Negotiated protocol must be the SDK's `LATEST_PROTOCOL_VERSION` (currently **`2025-11-25`**) — assert the value negotiated over the in-process transport equals it (a tripwire that reds if a future SDK bump changes the wire contract). _(The session value is negotiated, not a static server constant; assert the negotiated value, e.g. via the client after `initialize`.)_
5. **Contract snapshot + diff gate.** Over `InMemoryTransport`, call `tools/list`; serialize each tool's emitted JSON-Schema into a committed `packages/mcp/contracts/tools.snapshot.json`. **Note: SDK v1.x with zod 3.x emits JSON Schema draft-07 (not 2020-12)** — the snapshot will be draft-07; the snapshot test asserts the `$schema`/dialect so it can't silently change. Add a root `package.json` script `"contracts:check"` that regenerates the snapshot and `git diff --exit-code`s it. **Add a dedicated `contracts` job to `.github/workflows/ci.yml`** (single runner, Node 20: `pnpm install --frozen-lockfile` → `pnpm build` → `pnpm contracts:check`) — this PR creates the job; WS-B extends it. (This is a real `ci.yml` edit — the "turbo auto-covers it" only applies to build/typecheck/test, not the diff gate.)
6. **Tests (vitest, in-process + spawned).** `InMemoryTransport.createLinkedPair()` (`@modelcontextprotocol/sdk/inMemory.js`): assert `listTools` exposes the four tools + schemas; call each `scaffold_*` against a temp dir, assert files created + `structuredContent` validates against `outputSchema`; assert invalid input rejected; assert a read resource returns the discovery catalog; assert negotiated protocol == `LATEST_PROTOCOL_VERSION`. **Separately**, a stdout-hygiene test spawns the `stratawp-mcp` bin over real stdio and asserts every stdout line parses as JSON-RPC (the `InMemoryTransport` harness has no real stdout to inspect).
7. **Annotate the stale CHANGELOG line** (decision 12): a one-line note on `CHANGELOG.md:549` that the v0.4.0 AI feature was removed in v2.0 — defuses the "un-cancelled AI" optics of shipping `@stratawp/mcp`. (The broad CHANGELOG/README truth-up stays Phase 5.)

### WS-B — Contract-first enforcement (PR #2)

1. **Vendor a pinned `block.json` schema** (copy the canonical wp.org block schema into `packages/vite-plugin/schemas/block.schema.json` — do **not** validate against the live `https://schemas.wp.org/trunk/...` URL; a moving trunk makes CI non-deterministic). Validate every `src/blocks/**/block.json` (examples + `cli/templates`) against it plus a StrataWP superset (apiVersion 3; `name` prefix = theme namespace; `render` present iff dynamic) using the same schema lib as WS-A (`zod`) or `ajv`.
2. **Fix the ~15 mis-prefixed block names FIRST** (mandatory; the gate self-blocks otherwise): rename `forge-basic/*` and cross-theme `strata-basic/hero` to each theme's own namespace, in **both** `examples/**` and `packages/cli/templates/**`. **Before renaming, grep the repo for references to the old block names** (`forge-basic/`, `strata-basic/hero` in advanced/store) in templates/patterns so nothing renders broken; update any references. (The basic-theme block names changing must not break the axe/perf gates — content-only, but verify.)
3. **`theme.json` validation.** Validate each `theme.json` against the WP schema (structural). **Honesty:** the design-system bridge consumes only `color.palette` today — validate structure; do **not** claim a "design-token contract" beyond that.
4. **`block.json` → TS attribute codegen (ROADMAP item #3).** Extend the vite-plugin so each `block.json` emits a committed TS attribute-types file **next to the block** (NOT in gitignored `dist/`); consumed by the block's `edit.tsx`/`index.tsx` (**not** `save.tsx` — none exist). **Add a `typecheck` script to the example theme(s)** so the generated types + the `.tsx` that consume them are actually compiled in CI (today `examples/**` has no typecheck — without this, the codegen's value is unverified and the type-safety claim is hollow). Generated files must be prettier-formatted (`examples/**` is format-gated). A vite-plugin unit test asserts codegen output against a fixture `block.json`.
5. **Gate + injected-defect test.** Extend the `contracts` CI job (from WS-A) with: `contracts:validate` (schema validation of all block.json/theme.json) and `contracts:types:check` (regenerate codegen via `pnpm build`, then `git diff --exit-code` the generated types — sequence matters: build/codegen first, diff second). Include a **deliberately-broken `block.json` fixture** the validator must reject (mirrors the Phase-1 injected-defect bar). The gate is **required** (deterministic → can be required immediately once the tree is clean).

## 5. Key decisions (my call unless you redline)

1. **No embedded LLM, ever** — MCP protocol surface only; the user's own agent drives it.
2. **Option B** — WS-A (MCP server) then WS-B (contract enforcement), two sequenced PRs.
3. **Stable MCP SDK v1.x** (`@modelcontextprotocol/sdk ^1.29.0`, deep imports incl. `/inMemory.js`, zod ZodRawShape schemas) — **not** v2 alpha; explicit single-copy `zod ^3.25`.
4. **Zod schema IS the contract** — `z.infer` types, SDK emits JSON Schema **draft-07** (v1.x/zod-3 default), snapshot + `git diff --exit-code` gate (asserting the dialect); back-compat check before any breaking change.
5. **Refactor generator cores into pure, value-returning functions** shared by CLI + MCP — no `process.exit`/`console`/`fs`/`cwd` in the core; the command layer keeps IO. `block.ts` namespace parameterized.
6. **Resources from explorer `discoverAll()`**, read-only. The studio React UI is **not** revived; the divergent `BlockMetadata` unification is **out of scope** (resources use `ComponentInfo`).
7. **Headless data tools + hooks manifest deferred** (Option-C boundary). Destructive/credentialed CLI actions not exposed.
8. **Automated npm publish out of scope** — `@stratawp/mcp` ships via manual `pnpm release`; changesets init deferred. _(Redline to include.)_
9. **`block.json` validation strict, with mandatory name-prefix cleanup first** (~15 files incl. CLI-template copies) — then the gate is required day one (deterministic). _(Your call; flagged as real cleanup, not a baseline.)_
10. **Competitor AI bar accepted as inferred** (neither side ships MCP). _(Redline if you have a real scorecard row.)_
11. **Truth-up coordination** — Phase 4 adds no new claims; WS-B's codegen partially earns the JS half of `README.md:55`; full README truth-up stays Phase 5.

## 7. Risks & mitigations

- **SDK version trap:** stable single `@modelcontextprotocol/sdk ^1.29.0` + deep imports + `zod ^3.25` (ZodRawShape `inputSchema`); never the v2 alpha (`zod/v4`, `z.object()`). Verify a **single** hoisted zod (dual copies break the SDK's `instanceof`). Pin + verify on Node 18–24.
- **`--frozen-lockfile` reds until the lockfile is committed** — regenerating + committing `pnpm-lock.yaml` after adding the new deps is a required WS-A step, not an afterthought.
- **stdout poisoning:** all logs to stderr; the pure cores return values (no `console.*`); cover with the spawned-bin JSON-RPC-cleanliness test.
- **Pure-core refactor:** must not `process.exit`/write `fs`/`console` in the core or the MCP tool inherits process-killing/stream-poisoning behavior; the command layer retains all IO. No existing tests guard these commands — the new vitest suite is the only safety net.
- **JSON Schema dialect:** v1.x emits **draft-07**; the snapshot + test assert the dialect so it can't silently change (don't claim 2020-12).
- **Strict `block.json` gate self-blocks** on the ~15 stale names — fix them (and their template copies + references) before flipping the gate to required.
- **Generated-types verification gap:** `examples/**` has no `typecheck` today; without adding one, the codegen is unverified and the type-safety claim is hollow — WS-B adds the typecheck + a vite-plugin codegen unit test.
- **Contract gate placement:** it requires a real `ci.yml` job (not turbo auto-coverage); a single Node-20 runner (not the 4× matrix) to avoid wasteful/racy `git diff` runs.
- **Optics / consistency:** frame strictly as a protocol/contract surface the user's agent drives — never a revived AI SDK; annotate the stale CHANGELOG AI line; distinguish `@stratawp/mcp` (product) from `.mcp.json` (dev harness).
- **Codename / auto-scrub-hook:** neutral branch `feat/ai-readiness`; competitor = "Triple XXX"; **explicit `git add` only — never `git add -A`/`git add .`** (untracked, non-ignored `.mcp.json`/`.claude*`/`*.rvf`/`*.db`/`.stratawp-snapshots/` and the stray `docs/superpowers/plans/2026-04-26-*.md` whose filename embeds the competitor name would be swept in). Optionally git-ignore those traps (staged explicitly).
- **Type-safety claim coupling:** WS-B earns only the JS half of `README.md:55`; don't assert full PHP+JS type safety until Phase 5.
- **`docs/superpowers/*` is `.prettierignore`d** (spec/plan won't trip `format:check`); new `packages/mcp/**` `.ts`/`.json` and the generated types are lint/format-gated — format before commit.
